# Bob - Options Volatility Signal Generator (Phase 1)

## Overview
NestJS backend service for options volatility analysis based on the Volatility Risk Premium (VRP) strategy. Signal generation only - no trade execution.

## Architecture
- **Framework**: NestJS + TypeScript + Prisma ORM + PostgreSQL
- **Data Source**: Polygon.io API (options snapshots, historical bars, price data)
- **Strategy**: VRP = ATM_IV - HV_20; signals where vrp20 >= 95th pct AND iv_z >= 92.5th pct; top 5 selected

## Database Tables
- `universe`: 108 liquid stocks/ETFs (symbol, name, sector, active)
- `signal`: Daily computed features per symbol (atm_iv, hv_10/20/60, vrp_20, iv_z, rank, selected)
- `analysis_run`: Run metadata (run_id, status, timing, error tracking)
- `configuration`: System settings (thresholds, portfolio size, etc.)

## API Endpoints
- `GET /api/health` - Health check with DB status
- `GET /api/signals/latest` - Latest run signals
- `GET /api/signals/history` - Historical signals with pagination/filters
- `GET /api/analysis/runs` - Run history
- `GET /api/analysis/stats` - Aggregate statistics
- `GET /api/universe` - Symbol universe (filter by sector, active)
- `GET /api/config` - System configuration
- `POST /api/analysis/trigger` - Trigger analysis run (cron-protected with x-api-key header)

## Key Design Decisions
- Analysis runs via external cron job calling POST /api/analysis/trigger (no internal cron)
- CRON_API_KEY env var protects the trigger endpoint for automated runs
- Processes symbols in batches of 5 with 1.5s delay for rate limiting
- Stores signals in batches of 20 to avoid DB statement timeout
- IV z-score uses simplified approach: z = (IV - HV60) / (HV60 * 0.2)
- ATM IV: averages call+put IV for nearest-expiration strikes closest to current price (15-50 DTE)

## Environment Variables
- `DATABASE_URL` - PostgreSQL connection
- `POLYGON_API_KEY` - Polygon.io API key
- `CRON_API_KEY` - Secret for cron job authentication
