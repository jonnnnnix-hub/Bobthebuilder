# Autonomous Trading System (Phase 1.5)

## Overview

Phase 1.5 extends Bob from signal generation into fully autonomous paper-trading execution with model-driven decisions, risk controls, and live observability.

Pipeline:

1. Signal generated by Phase 1 scoring engine
2. Decision engine selects strategy + strikes + expiration + size
3. Dynamic risk evaluator approves/blocks
4. Execution service submits Alpaca paper order with retry logic
5. Position monitor updates live Greeks + P&L snapshots
6. Exit manager continuously evaluates and closes positions on trigger

## Core Modules

- `src/alpaca/`
  - credential-backed Alpaca REST client
  - account/positions/orders/options snapshots
  - order placement + position close

- `src/trading/decision-engine.service.ts`
  - Strategy selector: long/short call/put + straddle/strangle
  - Strike selector: delta-target and spread-aware
  - Expiration selector: term and liquidity weighted
  - Position sizer: score/confidence weighted to portfolio capital

- `src/trading/autonomous-risk.service.ts`
  - model-driven guardrails with dynamic symbol concentration, delta, and liquidity thresholds
  - persists `risk_metrics`

- `src/trading/autonomous-execution.service.ts`
  - scheduled worker (`@Interval(15000)`) for real-time execution
  - decision persistence (`trade_decision`)
  - order retry + logging
  - position sync + exit loop call

- `src/trading/exit-management.service.ts`
  - score-decay, stop-loss, profit-target, theta-decay exits
  - persists `exit_signal`
  - executes close via Alpaca

## Decision Examples

### Example A: High-confidence rich-vol signal

Input:
- Symbol: NVDA
- Composite score: 84
- Confidence: 0.80
- IV-Z: 2.1
- VRP: 0.18

Output:
- Strategy: `short_put`
- DTE: ~21–30 days
- Strike: near 0.20 delta put with tighter spread
- Size: larger notional from high score/confidence multiplier

### Example B: High score but illiquid chain

Input:
- Score: 82
- Confidence: 0.75
- Liquidity score from spread model: 0.10

Output:
- Trade blocked by risk module
- `trade_decision.risk_state = blocked`
- Reason logged: liquidity score below threshold

### Example C: Exit automation

Position monitored continuously. Exit triggered if any:
- unrealized P&L% <= -25%
- unrealized P&L% >= +35%
- DTE <= 3
- latest score normalized < 45

On trigger:
- `exit_signal` row created
- close submitted to Alpaca paper API
- execution logged in `trading_log`

## Deployment Notes (Fly.io)

Set secrets:

- `ALPACA_API_KEY`
- `ALPACA_API_SECRET`
- `ALPACA_PAPER_BASE_URL=https://paper-api.alpaca.markets`

Then deploy backend + frontend and verify:

1. `/api/trading/portfolio`
2. `/api/trading/positions`
3. `/api/trading/logs`

Run initial paper trades with small sizing and monitor logs/risk metrics.
